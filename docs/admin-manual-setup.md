# 管理员手动设置清单（note2 ④）

本文件列出需要管理员手动设置 / 维护的内容，供运营参考。后台地址：`/admin/`（需超级用户）。

## 1. 彩种基础数据（一次性 / 新增彩种时）

- 命令：`python manage.py seed_lotteries`
- 作用：写入双色球(ssq)、超级大乐透(dlt)、快乐8(kl8)、福彩3D(3d)、排列三(pl3) 的名称、分类、`rule_config` 等基础配置。
- 何时手动：首次部署、或新增/调整彩种规则时重跑（幂等，update_or_create）。

## 2. 开奖数据：抓取 + 发布（**每期都需关注**）

开奖数据是「抓取 → 草稿 → 手动发布」两步：

1. 抓取：`python manage.py crawl_draw`（抓全部）或 `python manage.py crawl_draw --code ssq`（指定彩种）。
   - 抓取结果写入 **草稿(draft)** 状态，前端**看不到**。
   - 注：定时调度尚未实现（note40 暂缓），目前需手动或自行配置 cron 执行。
2. 发布：进入后台 `开奖结果(DrawResult)` 列表 → 勾选要发布的记录 → 选择动作「**发布选中开奖记录**」。
   - 只有 `已发布(published)` 状态的记录才会出现在小程序里。

> 注意（note2 ③）：福彩3D / 排列三是**固定奖金玩法**，官方 `findDrawNotice` 接口对这两个彩种**不回填各奖级中出注数/奖金**，所以抓到的中奖明细为空属于数据源限制，并非 bug。

## 3. 玩法说明 / 奖级规则 / 通知 / 活动（按需录入）

后台 `玩法说明(PlayGuide)`，通过 `type` 区分用途，对应前端不同入口：

| type | 含义 | 前端入口 |
|---|---|---|
| `intro` | 玩法说明 | 玩法说明列表 |
| `rule` | 奖级规则 | 玩法说明页（直达规则） |
| `activity` | 活动公告 | 「通知」页 |
| `notice` | 通知公告 | 「通知」页 |

字段说明：
- `lottery`：关联彩种；留空 = 通用（所有彩种可见）。
- `title` / `content`：标题与正文（content 支持富文本）。
- `is_active`：是否上架（取消勾选即下架）。
- `sort`：排序（小的在前）。
- `is_important`：重要通知标记。
- `publish_at`：发布时间。

## 4. 微信登录配置（启用微信登录时）

- 环境变量：`WECHAT_APPID`、`WECHAT_SECRET`（见 `config/settings.py`）。
- 未配置时：登录走开发态 mock（匿名），不会真正调用微信。
- 配置后：小程序「微信登录」才能真正换取 openid/unionid。
- unionid 需小程序在微信开放平台绑定后才会返回。

## 5. 用户反馈查看

- 后台查看用户通过「我要反馈」提交的内容。

## 6. 协议文案（前端代码维护，非后台）

- 用户协议 / 隐私协议文案在前端 `lottery_frontend/src/utils/legal.js`，由开发同步，不在后台维护。
